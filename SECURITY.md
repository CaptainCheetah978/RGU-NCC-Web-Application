# Security Policy

## Supported Versions

The NCC RGU Cadet Management System is currently in active development. Security updates are provided for the following versions:

| Version | Supported          |
| ------- | ------------------ |
| 0.1.x   | :white_check_mark: |
| < 0.1   | :x:                |

As the application reaches a stable `1.0.0` release, this table will be updated to reflect long-term support (LTS) versions.

## Reporting a Vulnerability

We take the security of cadet data and our management system very seriously. If you discover a security vulnerability within this project, please **do not create a public issue**. Instead, follow these steps to report it responsibly:

1. **Contact the Maintainer:** Send a direct message or open a confidential security advisory via GitHub at [@CaptainCheetah978](https://github.com/CaptainCheetah978). Please include a descriptive summary, such as "[SECURITY REPORT] - Vulnerability Name".
2. **Include Details:** Provide as much information as possible. Helpful details include:
   - A description of the vulnerability and its potential impact (e.g., unauthorized data access, RLS bypass).
   - Steps to reproduce the issue, including any specific configurations or inputs required.
   - Potential fixes or suggestions, if any.
3. **Response Timeline:** You can expect an initial acknowledgment of your report within 48 hours. We will keep you updated on the progress of the investigation and the timeline for releasing a patch.
4. **Resolution:** If the vulnerability is accepted, we will work promptly to develop a fix, securely test it, and roll out a patch. Only after the patch is widely deployed will we consider a public disclosure or a CVE filing, ensuring our systems remain protected during the patching period.

Thank you for helping us keep the NCC RGU Cadet Management System secure!
